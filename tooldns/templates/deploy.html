⚙️ Required Environment Variables

Copy these into your platform's environment variables panel. Each platform has a "Variables" or "Environment" tab in their dashboard.

Variable What to put Required?
TOOLDNS_API_KEY Any secret string — e.g. td_mysecretkey123 Yes
TOOLDNS_APP_NAME Your brand name — e.g. MyAITools Optional
TOOLDNS_APP_TAGLINE Your tagline — e.g. AI Tools for Everyone Optional
TOOLDNS_CONTACT_EMAIL Your support email Optional
TOOLDNS_HOST 0.0.0.0 Yes
TOOLDNS_PORT 8787 (or whatever port your platform assigns) Optional
TOOLDNS_MODEL e.g. claude-sonnet-4-6 — for cost tracking Optional

🐳 Docker / VPS Step-by-Step

Run this on any Linux server (Ubuntu 22.04 recommended). Takes about 5 minutes.

Step 1 — Install Docker (skip if already installed)

curl -fsSL https://get.docker.com | sh

Step 2 — Clone your repo

git clone https://github.com/syedfahimdev/ToolsDNS.git
cd ToolsDNS

Step 3 — Create your .env file

cat > ~/.tooldns/.env <<EOF
TOOLDNS_API_KEY=td_your_secret_key_here
TOOLDNS_APP_NAME={{ app_name }}
TOOLDNS_APP_TAGLINE={{ app_tagline }}
TOOLDNS_CONTACT_EMAIL={{ contact_email }}
TOOLDNS_HOST=0.0.0.0
TOOLDNS_PORT=8787
EOF

Step 4 — Start with Docker Compose

docker compose up -d

Step 5 — Open the dashboard

http://YOUR_SERVER_IP:8787/ui

✅ That's it!

Your dashboard is live. Add a domain with Nginx or Caddy for HTTPS. Add tools from the Sources page.

🔒 Add HTTPS with Nginx + Let's Encrypt

Run these on your server after Docker is up. Replace yourdomain.com with your actual domain.

sudo apt install nginx certbot python3-certbot-nginx -y

# Create nginx config
sudo tee /etc/nginx/sites-available/toolsdns <<'NGINX'
server {
    listen 80;
    server_name yourdomain.com;
    location / {
        proxy_pass http://127.0.0.1:8787;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
    }
}
NGINX

sudo ln -s /etc/nginx/sites-available/toolsdns /etc/nginx/sites-enabled/
sudo nginx -t && sudo systemctl reload nginx

# Get SSL certificate (free)
sudo certbot --nginx -d yourdomain.com

🔓 Open Source — Safe for Contributors

Your brand config lives only in ~/.tooldns/.env which is gitignored. Anyone who forks or clones your repo gets clean defaults — they never see your API keys or brand settings.

What GitHub has (public)

  • Source code (AGPL-3.0)
  • Default config values
  • Templates + UI
  • No secrets, no brand

What stays on your server (private)

  • ~/.tooldns/.env — API keys + brand
  • ~/.tooldns/tooldns.db — your data
  • ~/.tooldns/skills/ — your skills
  • All gitignored automatically
